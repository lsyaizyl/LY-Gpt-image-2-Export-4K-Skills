# Skill：通用图片等比高清导出与清晰度增强工作流

## 1. 适用场景

当用户要求对任意图片进行高清化、清晰度增强、放大导出、修复后导出时，使用本 Skill。

适用于但不限于：

- 海报
- 插画
- 产品图
- 人像图
- 风景图
- 电商主图
- UI 截图
- 信息图
- 设计稿
- 文字海报
- 技术图
- 社交媒体配图
- AIGC 生成图
- 用户上传的普通照片

典型用户需求包括：

- “提高清晰度”
- “高清导出”
- “放大这张图”
- “保持比例导出高清版”
- “不要改变尺寸比例”
- “不要裁切”
- “不要拉伸”
- “导出更清晰的 PNG”
- “修复文字后再导出”
- “按原图比例生成大图”

---

## 2. 核心目标

本 Skill 的默认目标是：

> 在不改变原图宽高比例、不裁切、不拉伸、不破坏构图的前提下，提升图片清晰度并导出更高像素版本。

除非用户明确指定固定尺寸，否则不要强制套用固定尺寸，例如：

- 2160×3840
- 3840×2160
- 4096×4096
- 1920×1080

---

## 3. 基本原则

### 3.1 保持原图比例

默认必须保持原图宽高比例。

禁止：

- 强行拉伸
- 强行压缩
- 强行裁切
- 强行改成 9:16、16:9、1:1
- 把竖图改成横图
- 把横图改成竖图

除非用户明确要求改变比例。

---

### 3.2 先修复，再放大

如果图片存在以下问题，应先修复，再导出高清版：

- 中文乱码
- 错别字
- 伪文字
- 模糊小字
- 线条发虚
- 主体边缘破损
- 背景噪点
- 低清压缩痕迹
- AI 生成瑕疵
- 物体变形
- 人像五官异常
- 产品轮廓不干净

推荐顺序：

```text
识别问题 → 修复画面 → 确认比例 → 等比放大 → 锐化增强 → 验证尺寸 → 输出文件
```

---

### 3.3 区分“生成比例”和“像素尺寸”

图像生成或编辑工具通常更擅长控制：

- 画面比例
- 构图
- 风格
- 主体内容
- 清晰度倾向

但不一定能直接输出指定像素尺寸。

因此不要把：

```text
9:16
```

等同于：

```text
2160×3840
```

它们不是同一件事。

## 4. 判断用户需求类型

### 4.1 等比高清导出

当用户这样说时，默认使用等比高清导出：

- “提高清晰度”
- “高清一点”
- “放大一点”
- “保持比例”
- “不要变形”
- “不要裁切”
- “按原图比例”
- “输出高清版”
- “导出大图”

处理方式：

> 保持原图宽高比例，按倍数或目标长边等比放大。

---

### 4.2 固定尺寸导出

只有当用户明确说出固定尺寸时，才使用固定尺寸导出：

- “导出 2160×3840”
- “改成 1920×1080”
- “我要 3000×3000”
- “做成 16:9 横版”
- “裁成 1:1 方图”

处理方式：

> 按用户指定尺寸输出。必要时说明可能涉及裁切、留白或重构图。

---

### 4.3 修复后高清导出

当用户提到“乱码、模糊、错字、细节坏了、边缘不干净”等问题时：

> 先修复内容，再按原比例导出高清版。

## 5. 通用高清增强提示词

适用于绝大多数图片。

```text
请基于参考图制作一版更高清、更清晰的版本。

要求：
1. 保持原图主体、构图、宽高比例、色彩关系和整体风格不变。
2. 不裁切、不拉伸、不改变画面方向。
3. 提高清晰度、边缘锐度、细节密度和整体质感。
4. 主体边缘要干净自然，避免毛边、锯齿、残影。
5. 背景纹理、阴影、高光和材质细节要更清楚。
6. 如果图中有文字，文字应尽量清晰、规整、可读。
7. 不要随意改变主体身份、物体结构、服装、颜色、品牌标识或重要细节。
8. 不要增加无关元素。
9. 输出效果应像高清成品图，而不是模糊放大图。
```

## 8. 等比高清导出方式

### 8.1 默认推荐：按 2 倍放大

如果用户只说“高清一点”“提高清晰度”，且没有指定尺寸，默认可采用：

```text
2 倍等比放大
```

例如：

- 原图：941×1672
- 导出：1882×3344

---

### 8.2 更高清：按 3 倍放大

如果用户说：

- “更高清”
- “尽量高清”
- “适合打印”
- “大图导出”

可采用：

```text
3 倍等比放大
```

例如：

- 原图：941×1672
- 导出：2823×5016

---

### 8.3 按目标长边导出

如果用户没有指定宽高，但希望接近某个清晰级别，可以按长边控制：

- 长边 3000：普通高清
- 长边 4000：接近 4K 级大图
- 长边 5000+：海报级大图

逻辑：

```text
保持原图比例不变，将最长边放大到目标像素，另一边自动计算。
```

## 9. Python 等比导出代码模板

### 9.1 按倍数等比放大

```python
from PIL import Image, ImageFilter

src = "/mnt/data/input.png"
out = "/mnt/data/output_upscaled.png"

scale = 2.0

img = Image.open(src).convert("RGBA")
w, h = img.size

target_size = (round(w * scale), round(h * scale))

resized = img.resize(target_size, Image.Resampling.LANCZOS)

resized = resized.filter(
    ImageFilter.UnsharpMask(
        radius=1.4,
        percent=140,
        threshold=2
    )
)

resized.save(out, format="PNG", optimize=True)

print("原图尺寸：", (w, h))
print("导出尺寸：", target_size)
print("输出文件：", out)
```

---

### 9.2 按目标长边等比放大

```python
from PIL import Image, ImageFilter

src = "/mnt/data/input.png"
out = "/mnt/data/output_long_edge.png"

target_long_edge = 3840

img = Image.open(src).convert("RGBA")
w, h = img.size

scale = target_long_edge / max(w, h)
target_size = (round(w * scale), round(h * scale))

resized = img.resize(target_size, Image.Resampling.LANCZOS)

resized = resized.filter(
    ImageFilter.UnsharpMask(
        radius=1.4,
        percent=140,
        threshold=2
    )
)

resized.save(out, format="PNG", optimize=True)

print("原图尺寸：", (w, h))
print("导出尺寸：", target_size)
print("输出文件：", out)
```

---

### 9.3 保持比例并限制最大边，避免文件过大

```python
from PIL import Image, ImageFilter

src = "/mnt/data/input.png"
out = "/mnt/data/output_safe_hd.png"

target_long_edge = 3840
max_long_edge = 6000

img = Image.open(src).convert("RGBA")
w, h = img.size

target_long_edge = min(target_long_edge, max_long_edge)

scale = target_long_edge / max(w, h)
target_size = (round(w * scale), round(h * scale))

resized = img.resize(target_size, Image.Resampling.LANCZOS)

resized = resized.filter(
    ImageFilter.UnsharpMask(
        radius=1.4,
        percent=140,
        threshold=2
    )
)

resized.save(out, format="PNG", optimize=True)

print("原图尺寸：", (w, h))
print("导出尺寸：", target_size)
print("输出文件：", out)
```

## 11. 常见错误与规避

### 错误 1：把“高清”理解成固定尺寸

用户只说“高清”，不代表一定要：

- 2160×3840
- 3840×2160

默认应该：

> 按原图比例等比增强。

---

### 错误 2：拉伸图片

错误：

```text
原图是 941×1672，强行改成 2160×3840。
```

如果比例不同，会导致变形。

正确：

```text
保持原比例，按倍数或长边计算目标尺寸。
```

---

### 错误 3：修复文字后没有重新验证

文字修复后还要检查：

- 是否仍有乱码
- 是否出现伪文字
- 是否关键标题被改错
- 是否文字太小不可读

---

### 错误 4：没有告诉用户最终尺寸

必须明确告知：

- 原图尺寸
- 导出尺寸
- 是否保持比例
- 是否经过放大锐化
